One versioned synthetic activity record connects the three roles on the same device. It is continuity, not authentication. Implemented product loop
1

Student attempt

A learner chooses one fixed answer option. No name, score, timer, streak, or free text is required.

2

Activity observation

Deterministic code records correctness, hint use, and an activity-only observation. It does not infer ability.

3

Hint and follow-up

A project-authored visual hint supports another try. The follow-up result updates the same record.

4

Teacher review

The teacher sees attempts and hint use “in this activity,” then reviews the wording and result.

5

Bounded Strategy choice

Math and search choices become trusted activity configurations, not just labels.

6

Parent handoff

The parent receives one plain home activity. Raw prompts, transcripts, rank, and diagnosis stay out.

7

Changed next activity

The learner returns to a changed question, visual, trace, worked example, or explanation task.

The selected strategy returns to the learner as real behavior, so the learning link continues.

Versioned local learning record

attempt ID • synthetic profile • lesson ID • fixed option IDs • correctness • hint use • observation • possible-confusion code • review state • teacher strategy • timestamp

Deliberately excluded

real identity • school • location • contact details • health data • custom prompt • model transcript • rank • diagnosis
